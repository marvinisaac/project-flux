# Project Flux

A web-based, shared-world game based on an over-engineered backend.

Over-engineered aspects include (but is not limited to):
- Kubernetes
- API gateway
- (true) microservices
- message queue
- websocket

Some possible microservices include:
- authentication
- location
- combat
- loot drop
- character

Inspirations:
- Ragnarok
    - slot + card system
- Diablo series
    - loot system
- Dungeons and Dragons
    - emergent narrative

Timeline of Development
- Setup a barebones UI + backend utilizaing websockets
- Melee combat
